CH.2 METHODOLOGY

모듈화 접근법 — ❌ vs ✅

❌ 잘못된 방법

"인보이스 전체 자동화 코드 만들어줘"

→ 수백 줄 복잡한 코드

→ 오류 → 어디가 문제인지 모름

→ 수정 불가 → 처음부터 다시

결과: 포기 😫

✅ 올바른 방법 (모듈화)

업무를 단계별로 쪼갠다

→ 각 단계마다 짧은 코드 하나씩

→ 각 모듈 독립 테스트

→ 문제 모듈만 수정

결과: 완성 🎉

STEP 1/4

업무 모듈화 — 1. 업무를 단계별로 쪼개기

💡 "지금 내가 손으로 하는 작업을 순서대로 나열하면?"

현재 (손으로 하는 방식)

① 파일 열기 (거래처별 엑셀 5개)

② 이번 달 데이터 복사

③ 마스터 시트에 붙여넣기

④ 정렬 → ⑤ 합계 → ⑥ 날짜 통일

⑦ PDF 저장 → ⑧ 이메일 발송

소요: 약 2시간 · 매월 반복 · 실수 빈번

모듈화 결과

모듈 A: 파일 열기 + 데이터 읽기

모듈 B: 날짜 형식 통일

모듈 C: 마스터 시트 취합 + 정렬

모듈 D: 합계 계산

모듈 E: PDF 저장 + 이메일 발송

💡 실전 팁: A4 한 장에 지금 하는 업무를 있는 그대로 적어보는 것이 시작!

STEP 2/4

단계별 GPT 요청 — 2. 한 번에 하나씩!

핵심: 전체 요청 ❌ → 모듈 하나씩 요청 ✅

// 모듈 A 요청 예시 (GPT에게 보낼 프롬프트)

너는 10년 경력의 엑셀 VBA 전문가야.

다음 작업을 자동화하는 VBA 코드를 만들어줘:
- C드라이브 "정산" 폴더의 모든 .xlsx 파일을 열어
- 각 파일 Sheet1의 A열(날짜), B열(거래처명), C열(금액) 읽어와
- "취합" 시트에 순서대로 붙여넣어줘

조건: 헤더 제외 / 마지막 행까지만 / 한국어 주석 달아줘

💡 코드를 이해 못해도 OK — 실행해보고 잘 되면 다음 모듈로!

STEP 3/4

3. 오류 발생 시 대처법

🔴 VBA 실행 → 오류 발생 (예: "런타임 오류 1004")
▼
📋 오류 번호 + 메시지 그대로 복사
▼
🤖 GPT에게 붙여넣기: "이 오류 원인과 수정 코드 알려줘"
▼
✅ GPT가 즉시 원인 + 수정 코드 제공

STEP 4/4

4. 통합 VBA 코드 완성

Sub 월간정산_전체자동화()
  ' ===== 메인 실행 순서 =====

Call 모듈A_파일열기_데이터읽기' Step 1
Call 모듈B_데이터 형식_통일' Step 2
Call 모듈C_마스터시트_취합' Step 3
Call 모듈D_합계계산' Step 4
Call 모듈E_PDF저장_이메일발송' Step 5

  MsgBox "월간 정산 완료! " & Now()
End Sub
항목자동화 전자동화 후
속도약 2시간약 3분
정확도복붙 오류, 데이터 불일치거의 0%

CH.2 PROMPT

프롬프트 작성 원칙 3가지

1
역할을 부여하라
❌ "VBA 코드 만들어줘"
✅ "너는 10년 경력의 엑셀 VBA 전문가야. 다음 작업을 자동화하는 코드를 만들어줘."
2
맥락(Context)을 제공하라
❌ "정산서 만들어줘"
✅ "Sheet1의 A열에 날짜, B열에 거래처명, C열에 금액이 있어..."
3
출력 형식을 지정하라
❌ "설명해줘"
✅ "코드와 함께 각 줄이 무엇을 하는지 한국어 주석도 달아줘."

💡 프롬프트 = AI에 대한 업무 지시서(요건 정의서)

CH.2 TIPS

GPT+VBA 코딩 시 유의사항 5가지

  • ① 테스트 데이터로 먼저 실행 — 실제 데이터에 바로 적용하면 데이터 손실 위험
  • ② AI는 내 파일 구조를 모른다 — 시트명, 열 위치, 범위를 반드시 명시
  • ③ 오류 메시지는 그대로 GPT에 붙여넣기 — 즉시 원인과 수정 코드 제공
  • ④ 한 번에 너무 많이 요청하지 말 것 — 짧은 모듈이 훨씬 안정적
  • ⑤ 코드를 이해하려는 노력 — "이 코드가 무엇을 하는지 한줄씩 설명해줘" 습관화
