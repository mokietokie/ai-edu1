### SLIDE 4 | AI 에이전트란? 챗봇과 무엇이 다른가

| 구분 | 일반 AI 챗봇 | AI 에이전트 |
|------|-------------|------------|
| 동작 방식 | 질문하면 답변 (1회성) | 목표를 받고 스스로 계획 → 실행 → 검토 |
| 도구 사용 | 텍스트만 생성 | 이메일 발송, 파일 검색, 코드 실행, 웹 접속 |
| 자율성 | 사람이 매번 지시 | 단계별로 스스로 판단 후 실행 |
| 예시 | "이메일 초안 써줘" | "이번 주 미팅 일정 보고 참석자에게 자동 발송해줘" |

> "챗봇은 질문에 답하는 비서고, AI 에이전트는 스스로 판단하고 실행하는 직원이다."

### SLIDE 11 | VBA란 무엇인가?

**VBA (Visual Basic for Applications):**
- Microsoft 365 전 제품(Excel, Word, PowerPoint, Outlook)에 내장된 자동화 언어
- "엑셀에서 내가 손으로 하는 모든 작업을 기록하고 반복 실행하는 도구"
- **AI(GPT/Claude) + VBA = 코딩 몰라도 자동화 가능**

**왜 VBA인가?**
- 추가 설치 없음 — 이미 Excel에 있음
- 회사 IT 정책 승인 불필요
- Excel, Word, PPT, Outlook 모두 연동
- GPT가 코드 전부 생성 가능

### SLIDE 12 | VBA로 자동화 가능한 실무 작업 6가지

| # | 작업 유형 | 구체적 예시 | 예상 효과 |
|---|-----------|-------------|---------|
| 1 | **정기 보고서 자동 생성** | 월말 매출 데이터 → 정렬 → 차트 → PDF 저장 원클릭 | 2시간 → 3분 |
| 2 | **이메일 자동 발송** | 마감 알림, 미지급 거래처 안내 (Outlook 연동) | 반복 업무 제거 |
| 3 | **인보이스/정산서 자동화** | 데이터 입력 → 양식 생성 → 파일명 자동 저장 | 오류 최소화 |
| 4 | **데이터 정제 & 통합** | 여러 시트 합치기 + 중복 제거 + 포맷 통일 | 수작업 제거 |
| 5 | **마감 체크리스트** | 조건부 서식 + 항목별 마감 자동 알림 | 누락 방지 |
| 6 | **PPT/Word 자동 업데이트** | Excel 데이터 변경 시 보고서 자동 반영 | 이중 작업 제거 |

### SLIDE 13 | 모듈화 접근법 — ❌ vs ✅ 대조 비교

**슬라이드 레이아웃: 좌우 2분할 비교**

| ❌ 잘못된 방법 | ✅ 올바른 방법 (모듈화) |
|-------------|-------------------|
| "인보이스 전체 자동화 코드 만들어줘" | 업무를 단계별로 쪼갠다 |
| GPT가 수백 줄 복잡한 코드 생성 | 각 단계마다 짧은 코드 하나씩 요청 |
| 오류 발생 → 어디가 문제인지 모름 | 각 모듈 독립 테스트 → 오류 위치 즉시 파악 |
| 수정 불가 → 처음부터 다시 | 문제 모듈만 수정 → 나머지 그대로 사용 |
| 결과: 포기 | 결과: 완성 |

**자동차 조립 비유:**
```
엔진 모듈   +  변속기 모듈  +  차체 모듈   =  완성차
(데이터 읽기)  (날짜 정리)   (결과 출력)    (전체 자동화)
```

### SLIDE 14 | Step 1. 업무 모듈화 — 내 업무를 단계로 쪼개기

**핵심 질문:** "지금 내가 손으로 하는 작업을 순서대로 나열하면?"

**예시 — 월간 거래처 정산 업무 모듈화:**
```
[현재 손으로 하는 방식]
① 파일 열기 (거래처별 엑셀 파일 5개)
② 각 파일에서 이번 달 데이터 복사
③ 마스터 시트에 붙여넣기
④ 거래처별로 정렬
⑤ 합계 계산
⑥ 날짜 형식 통일 (각 파일마다 다름)
⑦ PDF로 저장
⑧ 이메일로 발송

→ 소요 시간: 약 2시간
→ 반복 주기: 매월 말
→ 실수 발생: 복붙 오류, 날짜 형식 불일치
```

**모듈화 결과:**
```
모듈 A: 파일 열기 + 데이터 읽기
모듈 B: 날짜 형식 통일
모듈 C: 마스터 시트 취합 + 정렬
모듈 D: 합계 계산
모듈 E: PDF 저장 + 이메일 발송
```

**실전 팁:** A4 한 장에 지금 하는 업무를 있는 그대로 적어보는 것이 시작

### SLIDE 15 | Step 2. 단계별 GPT 요청

### SLIDE 16 | Step 3. 각 모듈 독립 테스트

**테스트 환경 준비 (반드시 지킬 것):**

[테스트 전 체크리스트]

### SLIDE 17 | Step 4. 통합 VBA 코드 완성

**통합 방법 — 메인 프로시저 작성:**
```vba
Sub 월간정산_전체자동화()
    ' ===== 메인 실행 순서 =====
    
    Call 모듈A_파일열기_데이터읽기    ' Step 1
    Call 모듈B_날짜형식_통일          ' Step 2
    Call 모듈C_마스터시트_취합        ' Step 3
    Call 모듈D_합계계산               ' Step 4
    Call 모듈E_PDF저장_이메일발송     ' Step 5
    
    MsgBox "월간 정산 완료! " & Now()
End Sub
```

**GPT에게 통합 요청 시 프롬프트:**
```
앞서 만든 모듈 A, B, C, D, E 코드를 모두 붙여넣을게.
이것들을 하나의 메인 프로시저에서 순서대로 실행하도록
통합해줘. 각 모듈 실행 전후에 진행 상황을 상태바에
표시해주는 코드도 추가해줘.
```

**완성 후 실행 방법:**
```
Excel 열기 → [개발 도구] 탭 → [매크로] 버튼
→ "월간정산_전체자동화" 선택 → [실행]
→ 완료!
```

**자동화 전후 비교:**
| 항목 | 자동화 전 | 자동화 후 |
|------|---------|---------|
| 소요 시간 | 약 2시간 | **약 3분** |
| 실수 가능성 | 복붙 오류, 날짜 불일치 | **거의 0%** |
| 반복 가능성 | 매달 2시간 소요 | **버튼 한 번** |
| 야근 여부 | 월말 야근 발생 | **퇴근 가능** |

### SLIDE 18 | 프롬프트 작성 원칙 3가지

**원칙 1. 역할을 부여하라**
```
❌ "VBA 코드 만들어줘"
✅ "너는 10년 경력의 엑셀 VBA 전문가야. 다음 작업을 자동화하는 코드를 만들어줘."
```

**원칙 2. 맥락(Context)을 제공하라**
```
❌ "정산서 만들어줘"
✅ "Sheet1의 A열에 날짜, B열에 거래처명, C열에 금액이 있어.
    이 데이터를 Sheet2에 거래처별로 합산해서 정리하는 VBA 코드 만들어줘."
```

**원칙 3. 출력 형식을 지정하라**
```
❌ "설명해줘"
✅ "코드와 함께 각 줄이 무엇을 하는지 한국어 주석도 달아줘.
    그리고 테스트 시 주의사항도 알려줘."
```

> **핵심:** 프롬프트 = AI에 대한 업무 지시서.
> 신입직원에게 업무를 지시한다고 생각하고 구체적으로 작성할수록 좋은 결과가 나온다.

### SLIDE 19 | GPT+VBA 코딩 시 유의사항 5가지

**① 테스트 데이터로 먼저 실행**
- 실제 데이터에 바로 적용하면 데이터 손실 위험
- 항상 백업 파일 또는 더미 데이터로 먼저 테스트

**② AI는 내 파일 구조를 모른다**
- 시트명, 열 위치, 데이터 범위를 반드시 명시
- "Sheet1의 A2:A100에 거래처명이 있고..." 와 같이 구체적으로

**③ 오류 메시지는 그대로 GPT에게 붙여넣기**
- VBA 오류 발생 → 오류 번호 + 메시지 복사 → GPT에 그대로 붙여넣기
- GPT가 즉시 원인과 수정 코드를 제공

**④ 한 번에 너무 많이 요청하지 말 것**
- 긴 코드보다 짧은 모듈을 차례대로 만드는 것이 훨씬 안정적

**⑤ 코드를 이해하려는 노력 (포기하지 말 것)**
- "이 코드가 무엇을 하는지 한줄씩 설명해줘" 요청 습관화
- 이해 없이 붙여넣기만 하면 오류 시 속수무책

### SLIDE 20 | 업무 환경의 진화 & 3가지 시사점

**역사적 흐름:**
```
터미널 → GUI(마우스) → 엑셀/워드/PPT → 이메일/인터넷 → [지금] AI 에이전트
```
> 매번 새로운 도구가 등장할 때마다, 먼저 잘 쓰는 사람이 경쟁력을 가졌다.

**시사점 1. 아이디어만 있으면 누구나 만들 수 있다**
- 아이디어 → AI 에이전트 → 프로덕트/서비스
- 에이전트 구축 시간: **15~60분** (노코드 플랫폼)
- 해커톤 1위: 비개발자가 1주일 만에 허가 자동화 툴 완성

**시사점 2. 도메인 전문성이 새로운 경쟁력**
- 예전: 기술을 가진 사람이 문제를 해결했다
- 지금: **문제를 가진 사람이 기술을 쓰는 시대**
- IBM: "AI 창조 능력이 개발자를 넘어 일반 비즈니스 사용자로 이동"

**시사점 3. 매일의 불편함이 씨앗**
- 매일 반복하는 불편함 → 다음 자동화의 출발점
- C.H. Robinson: "미래의 직무는 AI 에이전트에게 지침을 주고 컨텍스트를 관리하는 일"
